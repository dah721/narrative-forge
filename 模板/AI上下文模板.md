00_上下文文件夹包含两个文件：
- 项目简报.md（AI每次会话的起点）

# 使用说明

## 两个文件的分工

| | 项目简报 | 当前阶段 |
|--|---------|---------|
| **内容性质** | 相对稳定的核心框架 | 动态变化的实时状态 |
| **更新频率** | 核心决策改变时 | 每次会话结束后 |
| **主要读者** | 新会话开始时的AI | 每次会话开始前的AI和创作者 |
| **长度** | 控制在可五分钟读完的范围 | 根据项目状态动态调整 |

## 初始化流程

新项目开始时，按以下顺序填写：

```
步骤一：填写项目简报的一至三部分（命题、终点、初始决策）
        ↓
步骤二：开始第一阶段工作
        ↓
步骤三：每次会话前更新当前阶段.md的"本次会话任务"部分
        ↓
步骤四：每次会话后更新当前阶段.md的其他部分
        ↓
步骤五：每次核心决策锁定后更新项目简报的"已锁定核心决策"部分
```

## 跨会话连续性检验

每次新会话开始时，用以下三个问题检验文档状态：

> 1. 只看文档，是否能知道项目目前在哪个阶段？
> 2. 只看文档，是否能知道已经锁定了哪些决策？
> 3. 只看文档，是否能知道本次会话需要处理什么？
>
> 三个问题都能被准确回答，文档状态合格。
> 任何一个无法回答，在开始实质性工作前先更新文档。
```
