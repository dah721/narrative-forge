# 写作创作系统

一套基于 Obsidian 的严肃叙事创作管理系统。它将创作过程拆解为六个阶段，通过结构化的文档体系和 AI 辅助流程，帮助创作者从一个模糊的感受出发，经过主题确立、形式决策、世界构建、人物设计、叙事架构，最终进入执行。

---

## 这个系统解决什么问题

创作长篇叙事作品时，常见的困境是：

- **主题失焦** — 写到中途才发现不知道自己想探讨什么
- **决策无记录** — 已经做出的设计决策散落在脑中或聊天记录里，无法被追溯
- **AI 辅助失控** — AI 参与创作时角色边界模糊，容易替代创作者做判断
- **跨会话断裂** — 每次打开文件都需要重新回忆项目状态

本系统的核心理念是：**创作的每一个阶段都应该产出可被检验的文档，而非只存在于创作者的头脑中。**

## 六阶段流程

| 阶段 | 名称 | 核心任务 | 完成标志 |
|:---:|------|---------|---------|
| 一 | **主题确立** | 提炼核心命题，验证内在张力，定义主题终点 | 命题可用一句话表达，且包含真实的张力 |
| 二 | **形式决策** | 选择媒介、结构策略、视角策略 | 形式选择有明确的主题理由 |
| 三 | **世界构建** | 建立规则，确保世界是主题的物理化实现 | 每个关键设定的主题功能已分析 |
| 四 | **人物设计** | 定义叙事功能与心理结构，通过删除测试 | 所有人物通过删除测试，工具性风险已评估 |
| 五 | **叙事架构** | 设计完整结构、信息节奏、读者体验 | 关键场景已识别，文学标准检验已通过 |
| 六 | **执行准备** | 建立写作规范与质量标准 | 规范可直接指导写作 |

**关键原则：**
- 前一阶段的核心决策锁定后才能进入下一阶段
- 可以返回前期阶段修改，但必须记录修改原因和影响
- 阶段不是强制流水线，跳过需要有明确理由

## AI 辅助的角色定义

本系统将 AI 定位为**分析工具**，而非创作者。

| AI 的功能 | AI 不应做的事 |
|----------|-------------|
| 分析主题的内在逻辑 | 替创作者决定主题方向 |
| 验证设计决策的自洽性 | 以"更有趣"或"更独特"为理由建议修改 |
| 识别执行风险（说教、廉价情感） | 改写创作者的叙事选择 |
| 提供多个可选方案 | 直接做出决策 |
| 检验文档的一致性 | 预测读者反应并以此作为修改依据 |

每次新会话开始时，AI 按标准流程读取 `项目简报.md` 和 `当前阶段.md`，从零建立对项目的理解。所有重要决策必须在会话结束前记录到文档中——口头确认但未记录的决策，在下次会话中将失效。

## 文档设计哲学

### 记录决策，而非描述内容

```
✗ 描述：世界是由血肉构成的，充满了可怕的怪物。

✓ 命题：血肉构成的世界是孩童原罪自我认知的物理化实现——
        从出生即有罪，身体本身即罪的证明。
```

### 三条检验原则

1. **功能优先** — 文档服务于创作决策，不是证明思考的深度
2. **可被检验** — 每一项内容应能回答"这对创作决策有什么具体影响"
3. **修改即历史** — 决策的修改过程与决策本身同等重要

## 快速开始

### 前置条件

- [Obsidian](https://obsidian.md/)
- 如需使用 AI 辅助流程：[Obsidian CLI](https://github.com/Obsidian-CLI/obsidian-cli)（可选）

### 使用方式

**1. 克隆仓库到本地，用 Obsidian 打开为 Vault：**

```bash
git clone <repo-url> 写作创作
```

在 Obsidian 中选择「打开文件夹作为仓库」→ 选择 `写作创作` 目录。

**2. 新建项目：**

在 `项目/` 下创建以项目名命名的文件夹，按照上方的项目结构建立子目录。使用 `模板/` 中的模板创建各阶段文档。

如果配合 AI 使用（如 Claude），AI 会自动完成项目结构建立和初始文档填写。

**3. 开始创作：**

从第一阶段（主题确立）开始。填写核心命题，验证内在张力，定义终点。满足完成标准后进入下一阶段。

**4. 维护文档：**

每次工作结束后更新 `当前阶段.md`，确保下次打开时能快速回到项目状态。

## 模板说明

| 模板 | 适用阶段 | 核心内容 |
|------|---------|---------|
| **AI上下文模板** | 新建项目时 | 项目简报结构 + 当前阶段结构 |
| **主题模板** | 第一阶段 | 命题提炼、哲学验证、终点定义、投资价值评估、核心载体 |
| **世界模板** | 第三阶段 | 叙事功能、核心规则、主题咬合、信息暴露、细节库 |
| **人物模板** | 第四阶段 | 叙事功能、心理结构、弧线设计、关系分析、可见性设计 |
| **叙事模板** | 第五阶段 | 结构策略、弧线设计、视角策略、信息设计、读者体验、关键场景 |

模板提供结构骨架，不是填空题。引导性说明（引用块）在实际文档中保留，作为持续的思考提示。

## 适用场景

本系统为**严肃叙事创作**设计，适用于：

- 小说（短篇、中篇、长篇）
- 视觉小说
- 剧本（影视、舞台）
- 游戏叙事

核心假设：创作者关心主题的深度与诚实性，而非仅仅关心情节的吸引力。
