# 写作创作系统

一套基于 Obsidian 的严肃叙事创作管理系统。它将创作过程拆解为六个阶段，通过结构化的文档体系和 AI 辅助流程，帮助创作者从一个模糊的感受出发，经过主题确立、形式决策、世界构建、人物设计、叙事架构，最终进入执行。

---

## 这个系统解决什么问题

创作长篇叙事作品时，常见的困境是：

- **主题失焦** — 写到中途才发现不知道自己想探讨什么
- **决策无记录** — 已经做出的设计决策散落在脑中或聊天记录里，无法被追溯
- **AI 辅助失控** — AI 参与创作时角色边界模糊，容易替代创作者做判断
- **跨会话断裂** — 每次打开文件都需要重新回忆项目状态

本系统的核心理念是：**创作的每一个阶段都应该产出可被检验的文档，而非只存在于创作者的头脑中。**

## 系统架构

```
写作创作/
├── 流程/                     ← 全局规范（适用于所有项目）
│   ├── 阶段指南.md           六阶段的定义、任务与完成标准
│   ├── 文档标准.md           文档书写的基本原则与格式要求
│   └── AI使用规范.md         AI 在各阶段的角色边界与禁止事项
│
├── 模板/                     ← 文档模板
│   ├── AI上下文模板.md       项目简报 + 当前阶段的结构模板
│   ├── 主题模板.md           核心命题、哲学基础、投资价值评估
│   ├── 世界模板.md           世界功能、规则、主题咬合分析
│   ├── 人物模板.md           心理结构、叙事功能、工具性检验
│   └── 叙事模板.md           结构策略、信息设计、读者体验设计
│
└── 项目/                     ← 所有创作项目
    └── [项目名]/
        ├── 00_上下文/
        │   ├── 项目简报.md   AI 每次会话的起点，核心决策汇总
        │   └── 当前阶段.md   实时状态、待处理问题、会话任务
        ├── 01_主题/          核心命题、主题终点、投资价值
        ├── 02_形式/          媒介决策、结构策略、视角策略
        ├── 03_世界/          世界规则、主题咬合、信息暴露
        ├── 04_人物/          每个人物一个文件
        ├── 05_叙事/          完整叙事结构、关键场景
        ├── 06_执行/
        │   └── 草稿/        实际写作内容
        └── 07_归档/
            ├── 探索记录.md   每次会话的关键发现与结论
            ├── 废弃路径.md   被否决的方向与理由
            └── 风险清单.md   已识别的叙事风险汇总
```

## 六阶段流程

| 阶段 | 名称 | 核心任务 | 完成标志 |
|:---:|------|---------|---------|
| 一 | **主题确立** | 提炼核心命题，验证内在张力，定义主题终点 | 命题可用一句话表达，且包含真实的张力 |
| 二 | **形式决策** | 选择媒介、结构策略、视角策略 | 形式选择有明确的主题理由 |
| 三 | **世界构建** | 建立规则，确保世界是主题的物理化实现 | 每个关键设定的主题功能已分析 |
| 四 | **人物设计** | 定义叙事功能与心理结构，通过删除测试 | 所有人物通过删除测试，工具性风险已评估 |
| 五 | **叙事架构** | 设计完整结构、信息节奏、读者体验 | 关键场景已识别，文学标准检验已通过 |
| 六 | **执行准备** | 建立写作规范与质量标准 | 规范可直接指导写作 |

**关键原则：**
- 前一阶段的核心决策锁定后才能进入下一阶段
- 可以返回前期阶段修改，但必须记录修改原因和影响
- 阶段不是强制流水线，跳过需要有明确理由

## AI 辅助的角色定义

本系统将 AI 定位为**分析工具**，而非创作者。

| AI 的功能 | AI 不应做的事 |
|----------|-------------|
| 分析主题的内在逻辑 | 替创作者决定主题方向 |
| 验证设计决策的自洽性 | 以"更有趣"或"更独特"为理由建议修改 |
| 识别执行风险（说教、廉价情感） | 改写创作者的叙事选择 |
| 提供多个可选方案 | 直接做出决策 |
| 检验文档的一致性 | 预测读者反应并以此作为修改依据 |

每次新会话开始时，AI 从 `项目简报.md` 和 `当前阶段.md` 重建对项目的理解。所有重要决策必须在会话结束前写入文档——口头确认但未记录的决策，在下次会话中将失效。

## 前置条件

### 必须安装

| 工具 | 用途 | 安装方式 |
|------|------|---------|
| [Obsidian](https://obsidian.md/) | 打开和编辑 Vault | 官网下载，免费 |
| [Obsidian CLI](https://obsidian.md/cli) | AI 通过命令行读写 Vault 文件 | Obsidian 设置 → 通用 → 开启 CLI 访问 |

> **Obsidian CLI** 是 Obsidian 官方提供的命令行接口。AI 使用它来自动建立项目结构、读取文档、写入更新。如果不使用 AI 辅助流程，只需要 Obsidian 本身。

### AI 客户端要求

本系统使用 [Claude](https://claude.ai) 配合 [Claude Code](https://claude.ai/code) 开发和测试。理论上兼容任何支持以下功能的 AI 客户端：

- **文件读写** — AI 能直接读取和修改 Vault 中的 `.md` 文件
- **命令行执行** — AI 能执行 `obsidian` CLI 命令
- **Skills / 自定义指令** — AI 能加载外部行为规范文件（`.md` 格式的 skill）

## 安装步骤

### 第一步：克隆仓库

```bash
git clone https://github.com/<your-username>/narrative-forge 写作创作
```

用 Obsidian 打开这个文件夹作为 Vault（「打开文件夹作为仓库」）。

### 第二步：开启 Obsidian CLI

在 Obsidian 中：`设置` → `通用` → 找到 **CLI 访问** → 开启。

开启后，在终端输入 `obsidian help` 确认可以正常使用。

### 第三步：安装 AI Skills

本系统包含两个驱动 AI 行为的 Skill 文件，需要安装到 AI 客户端的 skills 目录：

```
skills/
├── project-setup/          新建项目 skill
│   ├── SKILL.md            主行为规范
│   └── references/         参考文件（CLI 命令、模板格式）
└── enter-existing-project/ 进入已有项目 skill
    ├── SKILL.md
    └── references/
```

**安装方式（以 Claude Code 为例）：**

```bash
# 将 skills/ 目录复制到 Claude Code 的 skills 路径
cp -r skills/* ~/.claude/skills/
```

其他 AI 客户端请参考对应客户端的 custom instructions 或 system prompt 配置文档。

## 具体使用方法

### 新建项目

在 AI 会话中直接说：

> "开启一个新项目，项目名称是「XXX」"

AI 会自动完成：

1. 读取 `流程/` 下的三个规范文件，建立对系统的理解
2. 向你确认媒介类型和创作起点（四选一）
3. 在 `项目/[项目名]/` 下建立完整的文件夹结构
4. 基于模板创建 `项目简报.md`、`当前阶段.md` 和归档文件，并填入已知信息
5. 根据你的创作起点，引导进入对应的主题探索流程

### 继续已有项目

在 AI 会话中说：

> "继续「XXX」项目" 或 "回到「XXX」项目" 或 "我们上次聊到哪了"

AI 会自动完成：

1. 读取流程规范（每次都重读，因为 AI 无跨会话记忆）
2. 读取 `项目简报.md` 和 `当前阶段.md`，重建项目理解
3. 向你汇报当前项目状态、上次达成的决策、本次可能的起点
4. 等你确认本次会话任务后，才开始实质性工作

### 结束会话

每次工作结束时，告诉 AI：

> "结束项目" 或 "今天就到这里"

AI 会自动更新 `当前阶段.md`（进度、决策记录）和 `探索记录.md`（本次会话摘要），确保下次进入时状态准确。

### 不使用 AI 的手动模式

如果只想使用文档结构，不配合 AI：

1. 在 `项目/` 下手动创建项目文件夹，参照 [系统架构](#系统架构) 建立子目录
2. 从 `模板/` 复制对应模板到各阶段文件夹，重命名后填写
3. 每次工作结束后手动更新 `当前阶段.md`

## 模板说明

| 模板 | 调用时机 | 核心内容 |
|------|---------|---------|
| **AI上下文模板** | 新建项目时 | 项目简报结构 + 当前阶段结构 |
| **主题模板** | 进入第一阶段 | 命题提炼、哲学验证、终点定义、投资价值评估 |
| **世界模板** | 进入第三阶段 | 叙事功能、核心规则、主题咬合、信息暴露 |
| **人物模板** | 进入第四阶段，每人一个文件 | 叙事功能、心理结构、弧线设计、关系分析 |
| **叙事模板** | 进入第五阶段 | 结构策略、弧线设计、视角、信息设计、读者体验 |

模板提供结构骨架，不是填空题。引导性说明（引用块格式）在实际文档中保留，作为持续的思考提示。

## 文档设计哲学

### 记录决策，而非描述内容

```
✗ 描述：世界是由血肉构成的，充满了可怕的怪物。

✓ 命题：血肉构成的世界是孩童原罪自我认知的物理化实现——
        从出生即有罪，身体本身即罪的证明。
```

### 三条检验原则

1. **功能优先** — 文档服务于创作决策，不是证明思考的深度
2. **可被检验** — 每一项内容应能回答"这对创作决策有什么具体影响"
3. **修改即历史** — 决策的修改过程与决策本身同等重要

## 适用场景

本系统为**严肃叙事创作**设计，适用于：

- 小说（短篇、中篇、长篇）
- 视觉小说
- 剧本（影视、舞台）
- 游戏叙事

核心假设：创作者关心主题的深度与诚实性，而非仅仅关心情节的吸引力。
