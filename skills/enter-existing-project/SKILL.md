# 进入已有项目规范

> 核心原则：**读取优先于响应**。在完成必要读取之前，不开始实质性工作。
> 工具增强：使用 obsidian-cli 进行高效的文件搜索、读取和管理。
> 模板创建：需要新建文档时，使用 obsidian-cli 的 `template` 参数直接从模板创建。

## 零、根本前提

AI没有跨会话的记忆——上一次会话中达成的所有理解和口头决策，在本次会话中都不存在。**唯一可信的信息来源，是当前文档系统的实际内容。**

因此，进入已有项目本质上是从文档重建理解的过程，不是"继续上次的工作"。这就是为什么读取流程需要在任何实质性讨论之前完成。

---

## 二、读取流程

### 第一步：读取流程规范

这些文件路径固定，直接读取：

```bash
obsidian read path="流程/阶段指南.md"
obsidian read path="流程/文档标准.md"
obsidian read path="流程/AI使用规范.md"
```

| 文件 | 读取后需要明确的内容 |
|-----|------------------|
| `流程/阶段指南.md` | 当前项目所在阶段的任务定义、完成标准与常见错误 |
| `流程/文档标准.md` | 本次会话中文档书写与更新的基本规范 |
| `流程/AI使用规范.md` | 本次会话AI可以做什么、不可以做什么、会话结束时必须完成什么 |

如果文件不存在，告知创作者并等待确认路径。不在规范文件读取完成之前开始实质性工作。

---

### 第二步：确认模板库

```bash
obsidian ls path="模板"
```

确认以下模板文件存在：`主题模板.md`、`世界模板.md`、`人物模板.md`、`叙事模板.md`、`AI上下文模板.md`。

这些模板是所有阶段文档的建立基础。本次会话中需要新建文档时，将使用 obsidian-cli 的 `template` 参数直接从模板创建：

```bash
obsidian create path="项目/[项目名]/[阶段文件夹]/[文档名].md" template="[模板名]" silent
```

> 详细的模板调用流程和双向链接规范见 [references/template-workflow.md](references/template-workflow.md)——在实际需要创建文档时再读取。

---

### 第三步：定位并读取项目上下文

```bash
obsidian read path="项目/[项目名]/00_上下文/项目简报.md"
obsidian read path="项目/[项目名]/00_上下文/当前阶段.md"
```

| 文件 | 读取后需要明确的内容 |
|-----|------------------|
| `项目简报.md` | 核心命题、主题终点、已锁定的决策、AI总体定位、创作者工作偏好 |
| `当前阶段.md` | 当前阶段、本阶段进度、正在处理的内容、待处理优先级列表、本次会话任务 |

**读取项目简报后需要明确：** 核心命题、主题终点、哪些决策已锁定（不可重新讨论）、AI定位与禁止事项。

**读取当前阶段后需要明确：** 当前阶段、已完成/正在处理/待处理、本次会话任务（如已填写）。

---

### 第四步：读取与当前阶段相关的文档

只读取与当前阶段和当前任务直接相关的文档，不需要读取项目的所有文档。

```bash
obsidian ls path="项目/[项目名]/[当前阶段文件夹]"
```

| 当前阶段 | 需要读取的文档 |
|---------|--------------|
| 第一阶段 | `01_主题/` 下所有已建立文件 |
| 第二阶段 | `01_主题/`（已锁定）+ `02_形式/` 下所有已建立文件 |
| 第三阶段 | `01_主题/核心命题.md`（已锁定）+ `02_形式/`（已锁定）+ `03_世界/` 下所有已建立文件 |
| 第四阶段 | `01_主题/核心命题.md` + `03_世界/世界结构.md`（已锁定）+ `04_人物/` 下所有已建立文件 |
| 第五阶段 | 所有已锁定阶段的核心文档 + `05_叙事/` 下所有已建立文件 |
| 第六阶段 | 所有已锁定阶段的核心文档 + `06_执行/` 下所有已建立文件 |

如果当前阶段文件夹为空，告知创作者需要建立哪些文档，并说明将调用对应模板创建。

---

### 第五步：读取归档文件（按需）

以下归档文件不是每次都需要读取，只在特定情况下主动读取：

| 情况 | 需要读取的归档文件 |
|-----|----------------|
| 本次会话涉及风险评估 | `07_归档/风险清单.md` |
| 本次会话涉及已被否决的方向 | `07_归档/废弃路径.md` |
| 需要了解项目完整演变过程 | `07_归档/探索记录.md` |

---

## 三、状态确认

完成以上五步读取后，向创作者做简短的状态确认：

```
已读取：
- 流程规范（阶段指南/文档标准/AI使用规范）
- 模板库（已确认[X]个模板可用）
- 项目简报
- 当前阶段
- [其他读取的文档]

当前项目状态：
- 核心命题：[一句话]
- 当前阶段：第[X]阶段——[阶段名]
- 本阶段进度：[简述]
- 已锁定的核心决策：[条数]条

本次会话：
- 任务：[如果当前阶段.md中已填写则引用，否则询问]
- AI角色：[根据任务和阶段指南确认]

可以开始工作。
```

如果当前阶段.md中未填写本次任务，告知创作者并等待说明。没有明确任务就开始讨论容易偏离方向，浪费会话时间。

---

## 四、异常处理

遇到以下情况时，先告知创作者，等待确认后再继续。不自行判断以哪个版本为准。

### 文档内容与项目简报不一致

指出具体矛盾：项目简报记录的内容 vs 文档中显示的内容，请创作者确认以哪个版本为准。

### 当前阶段文件夹为空但项目简报显示该阶段已完成

说明可能是文档尚未建立或存在于其他位置，询问是否需要现在建立（将调用对应模板创建）。

### 当前阶段.md显示的阶段与项目简报不一致

指出差异，请创作者确认以哪个为准，确认后调整本次会话的工作框架。

### 项目简报的[待确认]标注过多

如果项目简报中有大量 `[待确认]` 但项目已推进到较后的阶段，建议先花几分钟更新项目简报，询问创作者是否同意。

### 风险清单中有与当前任务高度相关的风险

在状态确认时主动提示该风险及已记录的应对方向。

---

## 五、会话进行中的持续规范

### 关于已锁定决策

- 已锁定的决策不在本次会话中重新讨论
- 如果创作者提出与已锁定决策矛盾的方向，先指出矛盾，再询问是否需要正式返回该阶段进行修改
- 返回修改必须在文档中记录原因和影响范围

### 关于模板的使用

- 需要新建阶段文档时，使用 `obsidian create ... template="[模板名]" silent` 从模板创建
- 不从空白文件开始建立阶段文档
- 模板中的引导性说明（引用块格式）在实际文档中保留
- 创建后按 [references/template-workflow.md](references/template-workflow.md) 中的规范添加双向链接

### 关于文档更新

- 会话中做出的重要决策在达成后立即记录，不积压到会话结束
- 更新文档后确认是否有其他文档需要同步

---

## 六、会话结束时的标准操作

每次会话结束时，必须按顺序完成以下操作。详细的更新方式见 [references/template-workflow.md](references/template-workflow.md) 的"会话结束时的文档更新"部分。

1. **更新当前阶段.md** — 已完成、进行中、待处理、决策记录、清空本次任务、最后更新日期
2. **更新项目简报.md**（如有需要）— 新锁定的决策、修订的命题/终点、新风险、新文档索引、新偏好
3. **追加探索记录.md** — 本次任务、讨论内容、发现/推进、决策、未解决问题、下次建议起点
4. **更新阶段文档的修改记录**（如有修改）

---

## 七、完整检查清单

```
读取阶段：
□ 流程/阶段指南.md 已读取
□ 流程/文档标准.md 已读取
□ 流程/AI使用规范.md 已读取
□ 模板库已确认（或缺失已告知）
□ 项目简报.md 已读取
□ 当前阶段.md 已读取
□ 当前阶段相关文档已读取
□ 需要时已读取归档文件

状态确认阶段：
□ 已向创作者确认当前项目状态
□ 文档间的不一致（如有）已告知
□ 本次会话任务已明确
□ AI本次会话角色已明确

会话结束阶段：
□ 当前阶段.md 已更新
□ 项目简报.md 已更新（如有需要）
□ 探索记录.md 已追加本次会话记录
□ 阶段文档修改记录已更新（如有修改）
□ 双向链接已检查
```
