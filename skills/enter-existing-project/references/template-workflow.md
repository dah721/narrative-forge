# 模板调用与文档创建工作流

本文件定义了在进入已有项目的会话中，需要新建阶段文档时的详细操作步骤。

SKILL.md 中会在需要时指向本文件，无需提前全部读取。

---

## 模板调用流程

当需要使用模板创建新文档时，按以下步骤执行。

### 步骤一：确认模板存在

```bash
obsidian ls path="模板"
```

### 步骤二：使用模板创建文档（推荐方式）

直接使用 `template` 参数，这是最高效的方式：

```bash
obsidian create path="项目/[项目名]/[阶段文件夹]/[文档名].md" template="[模板名]" silent
```

- `template` 值是模板文件名（不含扩展名），如 `主题模板`、`世界模板`
- `silent` 防止文件在 Obsidian 中自动打开
- `overwrite` 可覆盖已存在的文件（慎用）

**示例：**

```bash
# 第一阶段
obsidian create path="项目/我的小说/01_主题/核心命题.md" template="主题模板" silent

# 第三阶段
obsidian create path="项目/我的小说/03_世界/世界结构.md" template="世界模板" silent

# 第四阶段（每个人物一个文件）
obsidian create path="项目/我的小说/04_人物/张三.md" template="人物模板" silent

# 第五阶段
obsidian create path="项目/我的小说/05_叙事/完整结构.md" template="叙事模板" silent

# 新增上下文文件
obsidian create path="项目/我的小说/00_上下文/新文件.md" template="AI上下文模板" silent
```

### 步骤三：回退方案

如果 `template` 参数创建失败，先读取模板内容再手动创建：

```bash
# 1. 读取模板内容
obsidian read path="模板/[模板名].md"

# 2. 用 Write 工具创建文档，内容为模板全文
```

当内容较长时（如项目简报模板），使用 Write 工具比 CLI 更可靠——CLI 命令长度超过约 200 字符可能会失败。

### 步骤四：添加双向链接

新文档创建后，追加相关链接：

```bash
obsidian append path="项目/[项目名]/[阶段文件夹]/[文档名].md" content="\n\n---\n相关链接：[[项目简报]] [[当前阶段]]" silent
```

也可使用 `file` 参数简化路径：

```bash
obsidian append file="[文档名]" content="\n\n---\n相关链接：[[项目简报]] [[当前阶段]]" silent
```

### 步骤五：验证

```bash
obsidian ls path="项目/[项目名]/[阶段文件夹]"
```

## 各阶段对应模板

| 当前阶段 | 需要创建的文档 | 调用的模板 |
|---------|--------------|-----------|
| 第一阶段 | 01_主题/核心命题.md | 模板/主题模板.md |
| 第三阶段 | 03_世界/世界结构.md | 模板/世界模板.md |
| 第四阶段 | 04_人物/[人物名].md | 模板/人物模板.md |
| 第五阶段 | 05_叙事/完整结构.md | 模板/叙事模板.md |
| 任意阶段 | 新增上下文文件 | 模板/AI上下文模板.md |

## 双向链接规范

创建或修改项目文档时，维护以下双向链接关系：

### 各文档的链接目标

1. **新建的阶段文档** → 链接到 `项目简报` 和 `当前阶段`
2. **项目简报** → 链接到本次新建的重要文档
3. **当前阶段** → 链接到本阶段已锁定的决策文档
4. **探索记录** → 链接到本次会话涉及的所有文档

### 添加方式

**短内容（obsidian CLI）：**

```bash
obsidian append path="项目/[项目名]/01_主题/核心命题.md" content="\n\n---\n相关链接：[[项目简报]]" silent
```

**长内容或需要精确位置时：** 使用 Edit 工具在文档适当位置插入。

### 会话结束前检查清单

- [ ] 新建的阶段文档是否链接到项目简报和当前阶段？
- [ ] 项目简报是否链接到本次新建的重要文档？
- [ ] 当前阶段是否反映了所有已锁定的决策文档？
- [ ] 探索记录是否链接到本次讨论涉及的文档？

## 会话结束时的文档更新

### 更新当前阶段.md

更新以下部分：
- 已完成（新增本次完成的内容）
- 进行中（更新当前进展状态）
- 待处理（调整优先级，新增或删除条目）
- 最近的重要决策记录
- 本次会话任务（清空，供下次会话填写）
- 当前最需要警惕的风险（如有变化则更新）
- 最后更新日期

**短内容用 obsidian CLI：**

```bash
obsidian append file="当前阶段" content="- [新增待处理项]" silent
obsidian insert path="项目/[项目名]/00_上下文/当前阶段.md" after="## 已完成" content="\n- [本次完成的任务]" silent
```

**长内容用 Edit 工具。**

### 更新项目简报.md（如有需要）

以下情况需要更新（使用 Edit 工具）：

| 情况 | 更新的部分 |
|-----|-----------|
| 锁定了新的核心决策 | 三、已锁定的核心决策 |
| 核心命题被修订 | 一、核心命题 |
| 主题终点被修订 | 二、主题终点 |
| 发现新的重要风险 | 五、主要风险提示 |
| 新建了重要文档 | 六、关键文档索引 |
| 新的工作偏好 | 八、创作者偏好备注 |

### 追加探索记录.md

使用以下格式追加到探索记录末尾：

```markdown
## [日期] 会话记录

**本次任务：**

**讨论内容：**

**主要发现或推进：**

**达成的决策：**

**未解决的问题：**

**下次会话的建议起点：**
```

**短内容用 `obsidian append`，长内容用 Edit 工具。**

### 更新阶段文档修改记录

如果本次会话修改了阶段文档，在该文档的修改记录部分追加：

```markdown
| [日期] | [修改内容] | [修改原因] | [影响的其他文档] |
```
