# 新建项目 Skill

在创作者发起新项目时，建立可以立即投入工作的项目结构。

核心原则：**只建立当前需要的内容，不预填充尚未讨论的部分。**

---

## 工作流程总览

```
准备工作 → 信息收集 → 建立文件结构 → 填写初始文档 → 根据起点引导进入对应流程
```

---

## 零、进入项目前的必要准备

新会话的 AI 对这个创作体系一无所知，所以需要先读取规范建立理解，再与创作者互动。

### 步骤一：读取流程规范

这些文件路径固定，直接读取：

```bash
obsidian read path="流程/阶段指南.md"
obsidian read path="流程/文档标准.md"
obsidian read path="流程/AI使用规范.md"
```

| 文件 | 读取后需要明确的内容 |
|-----|-------------------|
| 阶段指南.md | 六个阶段的定义、任务、完成标准与常见错误 |
| 文档标准.md | 文档书写的基本原则、命题与描述的区分方式 |
| AI使用规范.md | AI的根本定位、各阶段的角色边界、禁止事项 |

如果文件不存在，告知创作者并等待确认路径。

### 步骤二：确认模板库

```bash
obsidian ls path="模板"
```

确认以下模板存在：`主题模板.md`、`人物模板.md`、`世界模板.md`、`叙事模板.md`、`AI上下文模板.md`。

模板是文档的起点，不是约束——使用模板意味着在模板结构基础上填写，不从空白文件开始。模板中的引导性说明（引用块格式）在实际文档中保留。

| 模板 | 调用时机 |
|-----|---------|
| AI上下文模板.md | 新建项目时立即调用（建立项目简报与当前阶段） |
| 主题模板.md | 进入第一阶段，创建核心命题时 |
| 世界模板.md | 进入第三阶段，创建世界结构时 |
| 人物模板.md | 进入第四阶段，为每个人物创建文件时 |
| 叙事模板.md | 进入第五阶段，创建完整结构时 |

### 步骤三：向创作者确认准备完成

```
流程规范与模板库已读取完毕。

已确认：
- 六阶段流程规范
- 文档书写标准
- AI在本项目中的角色边界
- 可用模板：主题/人物/世界/叙事/AI上下文

可以开始新建项目。请告知项目名称与媒介类型。
```

---

## 第一步：信息收集

在建立任何文件之前，向创作者确认以下必要信息。使用 AskUserQuestion 工具一次性收集。

### 必须确认

1. **项目名称** — 可以是暂定名，之后可以修改
2. **媒介类型** — 小说、视觉小说、剧本、游戏、其他
3. **当前起点** — 决定后续引导方式：
   - A. 只有一个模糊的感受或问题意识
   - B. 已经有了一个比较清晰的主题方向
   - C. 已经有了人物或故事的构想
   - D. 已经有了部分内容，需要整理进系统

### 可选信息（不主动询问，创作者提到则记录）

- 参照文本或灵感来源
- 预期体量（短篇/中篇/长篇）
- 创作周期预期
- 特殊的创作限制或偏好

---

## 第二步：建立文件结构

确认项目名称后，在 `项目/` 下建立以下结构：

```
项目/[项目名]/
├── 00_上下文/
│   ├── 项目简报.md          ← 立即创建并填写
│   └── 当前阶段.md          ← 立即创建并填写
├── 01_主题/
├── 02_形式/
├── 03_世界/
├── 04_人物/
├── 05_叙事/
├── 06_执行/
│   └── 草稿/
└── 07_归档/
    ├── 探索记录.md          ← 立即创建（空文档）
    ├── 废弃路径.md          ← 立即创建（空文档）
    └── 风险清单.md          ← 立即创建（空文档）
```

**建立原则：** 只建立文件夹，不预建内容文件（内容文件在对应阶段开始时才建立）。例外是 `00_上下文/` 和 `07_归档/` 下的文件需立即创建。

> 具体的 CLI 命令和回退方案见 [references/cli-operations.md](references/cli-operations.md)——按文件中的步骤执行即可。

---

## 第三步：填写初始文档

使用 `obsidian create ... template="AI上下文模板" silent` 基于模板创建上下文文件，再用 Edit 工具填充项目特定信息。

### 创建清单

1. **项目简报.md** — 填入项目名称、媒介类型、今天日期；未知部分标注 `[待确认]`
2. **当前阶段.md** — 填入媒介类型和今天日期，设定为第一阶段（主题确立）
3. **探索记录.md** — 创建空文档（只有 frontmatter 和标题）
4. **废弃路径.md** — 创建空文档
5. **风险清单.md** — 创建空文档

模板详见 [references/templates.md](references/templates.md)。CLI 操作详见 [references/cli-operations.md](references/cli-operations.md)。

创建后为项目简报和当前阶段互相添加双向链接。

---

## 第四步：根据起点引导进入对应流程

文件结构建立完成后，根据创作者选择的起点，用对应的引导语开启后续对话。

### 起点 A：只有模糊的感受或问题意识

> 文件结构已建立。
>
> 你现在有的是一种感受或一个问题意识——
> 在我们开始正式讨论之前，能否用一两句话描述它？
> 不需要完整，不需要准确，只是目前心里有的东西。

**后续：** 从感受出发，通过对话逐步提炼核心命题。命题确立的标准是创作者能**主动**用一句话说出它，而不是被动确认 AI 的表述——这个区别很重要，因为被动确认往往意味着创作者还没有真正内化这个命题。

### 起点 B：已有比较清晰的主题方向

> 文件结构已建立。
>
> 请说说你目前对主题的想法。
> 尽量具体——不只是主题是什么，
> 还包括：你觉得这个主题有意思的地方是什么？

**后续：** 直接进入主题分析——识别命题的内在张力、验证创作者直觉与命题逻辑的一致性、确认主题终点、评估投资价值。

### 起点 C：已有人物或故事构想

> 文件结构已建立。
>
> 你已经有了人物或故事的构想——
> 在我们讨论这些构想之前，我需要先了解：
> 这个故事想探讨什么？或者说，是什么让你想写这个故事？

**后续：** 即使创作者已有人物或故事，仍然从主题开始。人物和故事构想是主题的潜在载体，但在主题确立之前，无法判断这些构想是否真正服务于创作者想探讨的东西。

### 起点 D：已有部分内容，需要整理进系统

> 文件结构已建立。
>
> 你已经有了一些内容——
> 在整理之前，我需要先了解目前的状态：
> 1. 已经有了什么？（主题、世界、人物、情节、文字，各有多少）
> 2. 你认为目前最需要解决的问题是什么？
> 3. 整理进系统的主要目的是什么？
>    （澄清思路、发现问题、准备继续推进、还是其他）

**后续：** 根据回答判断从哪个阶段开始：

```
主题是否已经清晰可以被一句话表达？
├── 否 → 从第一阶段开始，先确立主题
└── 是 → 主题文档先行建立
          ↓
          世界和人物是否有足够的设计？
          ├── 否 → 对应阶段待填充
          └── 是 → 逐阶段整理，同时做一致性检验
```

整理时同步检验已有内容与核心命题的一致性。不一致之处先记录，整理完成后统一评估。

## 常见情况处理

这些情况在项目建立后的主题确立阶段经常出现。处理的共同精神是：**不强制要求，而是通过提问自然引导回主题。**

### 创作者想跳过主题直接讨论情节

不拒绝，而是问："这个情节想制造的是什么样的读者体验？不是情节本身发生了什么，而是读者经历这个情节时，内部应该发生什么？"——这通常会自然引回主题讨论。

### 创作者的"主题"实际上是情节概要

如果可以用"一个X的人经历了Y最终Z"来概括，这是情节概要。引导："你描述的是故事会发生什么——这个故事探讨的核心问题是什么？读完之后，你希望读者带走什么？"

### 创作者的"主题"是一个道德结论

如"爱是最大的力量"。不否定，而是挖掘张力："是什么让这个方向在你看来值得探讨？它本身有什么让你感到困难或有趣的地方？"

### 创作者在第一阶段想讨论太多

第一阶段只需完成主题确立。人物、世界、情节如果自然浮现，记录到探索记录中，但不展开："这个想法很有意思，我们先记录下来。主题确立之后，它是否服务于我们想探讨的东西，到时候会更清楚。"

### 创作者对框架有异议

框架服务于创作者，不是约束创作者。回复："完全可以调整。能告诉我你觉得哪里不合适吗？是太繁琐、还是方向不对、还是其他原因？"

## 第一次会话结束时的标准操作

无论起点如何，第一次会话结束时需要完成以下更新。具体的 CLI 命令见 [references/cli-operations.md](references/cli-operations.md) 的"第一次会话结束时的文档更新"部分。

1. **当前阶段.md** — 更新"已完成"、"进行中"、"最近的重要决策记录"、"待处理"
2. **探索记录.md** — 追加本次会话记录（格式见 [references/templates.md](references/templates.md)）
3. 如果本次会话确立了核心命题或主题终点 → 立即更新**项目简报.md**，将 `[待确认]` 替换为实际内容

## 阶段推进时的模板调用

进入新阶段需要创建阶段文档时，使用 `obsidian create ... template="[模板名]" silent` 从模板创建。

```
进入第一阶段 → template="主题模板"   → 01_主题/核心命题.md
进入第三阶段 → template="世界模板"   → 03_世界/世界结构.md
进入第四阶段 → template="人物模板"   → 04_人物/[人物名].md（每人一个）
进入第五阶段 → template="叙事模板"   → 05_叙事/完整结构.md
```

创建后添加双向链接（链接到项目简报和当前阶段）。如果模板创建失败，先读取模板内容再用 Write 工具手动创建。

> 完整的模板调用流程、回退方案、双向链接规范见 [references/cli-operations.md](references/cli-operations.md)。

## AI 角色定义

在项目建立和主题确立阶段，AI 的角色是：

- **哲学分析者** — 验证逻辑自洽性
- **风险识别者** — 识别执行风险

AI 始终不应做的事（这些边界的存在是因为创作的核心价值在于创作者自身的判断和表达）：
- 替代创作者确定主题方向
- 以文学标准之外的标准评判内容
- 在创作者框架外强加新框架

```
□ 模板库已确认存在
□ 已向创作者确认准备完成

文件结构：
□ 项目目录和所有子文件夹已建立
□ 项目简报.md 已基于模板创建并填写
□ 当前阶段.md 已基于模板创建并填写
□ 归档文件（探索记录/废弃路径/风险清单）已创建
□ 双向链接已添加

信息确认：
□ 项目名称已确认
□ 媒介类型已确认
□ 创作者起点已判断
□ 对应流程已启动

会话结束：
□ 当前阶段.md 已更新
□ 探索记录.md 已追加会话记录
□ 项目简报.md 已更新（如有新决策）

```
